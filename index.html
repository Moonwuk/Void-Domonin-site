// О ПРОЕКТЕ

Командуй флотами.
Захватывай сектора.
Доминируй в пустоте.

Асинхронная real-time стратегия: мир вращается непрерывно, игроки действуют в своём темпе. Не пошаговая — темп как вдохновение, все механики оригинальные.

Void Dominion — это тёмный космос, где сотни флотов и дивизий сходятся в непрерывной войне за сектора галактики.

Выйдя из игры, ты не ставишь мир на паузу — он продолжает вращаться. Твоя экономика копит ресурсы, флоты летят по приказам, данным час назад, а ИИ-«Хранитель» подстрахует твоё место, пока ты спишь.

Управляй 5 ресурсами, строй станции и здания, собирай флоты и наземные дивизии из шаблонов. Война идёт в две фазы: орбитальный бой → десант. Артиллерия бьёт через лейны с дугами снарядов, герои дают ауры, учёные ускоряют исследования.

Дипломатия — по согласию: мир, война, пакт, союз. Шпионаж с контрразведкой. Сессионная биржа. Мета-прогрессия командира с тремя ветками прокачки между матчами.

// СЕТЕВОЙ РЕЖИМ

  • Скирмиш против ИИ — одиночная партия
  • Командный бой — стороны-союзники против ИИ
  • Онлайн-матч до 10 живых игроков по позывным
  • Режимы: FFA / 5v5 / 2v2
  • Лобби, seat-lock-билеты, durable Postgres
  • Матч переживает рестарт сервера
  • ИИ-подмена отключившихся игроков
  • Эфемерные ally-пинги на карте
// МЕХАНИКИ

22 модуля поверх микроядра

Каждая механика — отдельный подключаемый модуль. Ядро не трогается, новая возможность = новый модуль + данные.

M01

Real-time 24/7

Мир идёт непрерывно и круглосуточно. Сервер-авторитет с offline-планировщиком: матчи живут без подключённых игроков.

M02

5 ресурсов и экономика

Сбор, переработка, склады. Постройки и станции. Сессионная биржа для обмена между игроками.

M03

Флот + наземные дивизии

Сборка шаблонов из пехоты и танков с превью характеристик. Транспорт перевозит десант между секторами.

M04

Двухфазный захват

Орбитальный бой → десант. ПВО и бомбардировка. Перехват на лейнах. Walk-in захват необоронённого нейтрального сектора.

M05

Дальнобойная артиллерия

Standoff-огонь и barrage-приказы через лейны. Дуги снарядов. Угроза на расстоянии без прямого контакта.

M06

Герои и учёные

Герои дают ауры и респаун. Учёные ведут исследования. Дерево технологий с открытием веток.

M07

Дипломатия по согласию

Стойки: мир / война / пакт / союз. Офферы с одобрением. Боты принимают решения по своим интересам.

M08

Шпионаж и контрразведка

Разведка секторов и чужих флотов. Контрразведка ловит чужих шпионов. Туман войны с памятью и radar-сигнатурами.

M09

«Хранитель»

Делегируй место ИИ на время сна. Мир идёт 24/7 — твой сектор не умрёт, пока ты не зашёл.

M10

Мета-прогрессия командира

XP между матчами. Три ветки прокачки. Корпорации с ролями, заявками, аудитом и войнами AvA.

M11

Дерево технологий

Data-driven ветки исследований. Учёные ускоряют открытие. Открывает новые юниты, здания и способности.

M12

Onboarding и чат

Гайдовый первый матч, туры-подсветки, поисковый справочник, чеклист целей. Сессионный чат и пинги.

// АРХИТЕКТУРА

Принципы разработки

Void Dominion построен на жёстких архитектурных принципах. Они обеспечивают детерминизм, античит и устойчивость к десинку.

01 // DATA-DRIVEN

Игровые объекты — в JSON

Игровые объекты описаны в JSON-данных. Ядро лишь выполняет правила. Контент отдельно от кода — новые фракции и юниты без правки движка.

02 // MICROKERNEL

Микроядро + модули

Маленькое неизменяемое ядро, всё остальное — подключаемые модули. Они общаются только через шину: события, хуки, реестр возможностей.

03 // DETERMINISM

Детерминизм

Ядро — чистая функция: одинаковый вход → одинаковый выход. Seeded RNG, время как параметр. Replay боёв, предпросмотр на клиенте, античит.

04 // SERVER-AUTHORITY

Сервер — авторитет

Клиент шлёт намерение, сервер решает. Это основа античита: состояние мира строится только на подтверждённых сервером действиях.

05 // FOG-AS-BOUNDARY

Туман войны — граница безопасности

Сервер физически не отправляет невидимое. Не «шлёт всё и прячет на клиенте», а проекция visibleState — туман как часть security-модели.

06 // FAIL-SECURE

Безопасный отказ

Любая ошибка ведёт к безопасному отказу, не к пропуску. Лучше оборвать сессию, чем допустить несогласованное состояние мира.

// СТАТУС РАЗРАБОТКИ

Этапы проекта

Каркас, ядро, слой действий, сервер и клиент — где проект находится сейчас.

Этап 0 // Каркас
ГОТОВО
Монорепа, пакеты, CI, TypeScript-everywhere, базовая инфраструктура.
Этап 1 // Ядро
ГОТОВО
Микроядро + шина + хуки + манифест, seeded RNG, real-time advanceTo, экономика, бой, дипломатия, шпионаж, герои, туман войны. Покрыто тестами.
Этап 2 // Слой действий
ГОТОВО
Envelope-валидация, zod-схемы payload'ов, сессии, идемпотентность, clientSeq. Подключается на сервере через GATE=1.
Этап 3 // Сервер
В РАБОТЕ
WS-мультиплеер + дельта-sync + туман на рассылке + Postgres-персистентность + offline-планировщик + мульти-матч реестр + аккаунты JWT. Впереди — OIDC, мульти-процесс.
Этап 4 // Клиент
В РАБОТЕ
Играбельный клиент в prototype/ (браузер + Capacitor → APK, RU/EN, мобильный UI). packages/client — transport-adapter + Vite-shell с живой картой.
